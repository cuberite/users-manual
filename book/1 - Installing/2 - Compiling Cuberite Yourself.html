Compiling yourself takes longer and is more involved, but on modern processors can lead to a speed increase of up to 1.5 to 3x. If your operating system or hardware is not officially supported, compiling may be the only way to run Cuberite.

  • *nix (Debian, Ubuntu, etc.) users can use the automatic compile.sh script, which is described below.
  • All other operating system users (Windows, Mac OS, etc.), and Linux users who are intereted in manually compiling should see COMPILING.md in the main repository.

Compile.sh can also update your server. Re-running Compile.sh from the code directory automatically updates your server by fetching the latest code and then re-compiling.

Compiling takes a while on slow hardware like Raspberry Pis, up to several hours in some cases. However, subsequent compiles will be faster.
